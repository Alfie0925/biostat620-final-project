Overview

This page documents the full data pipeline behind the main report. The analysis uses five consecutive years of CDC Behavioral Risk Factor Surveillance System (BRFSS) public-use survey data, then trims and harmonizes those files into a smaller analytic dataset focused on frequent mental distress (FMD), geography, demographic factors, and selected behavioral and psychosocial correlates.

The preparation code used to build the analytic dataset is available in the project repository at prep_brfss_subset.R. The rendered website reads the resulting analytic file from brfss_subset.rds.

Data Background

This project uses multiple years of data from the Behavioral Risk Factor Surveillance System (BRFSS), a large national health survey conducted by the Centers for Disease Control and Prevention (CDC) in collaboration with U.S. states and participating territories. Established in 1984, BRFSS collects standardized, state-specific information on health-related risk behaviors, chronic conditions, health care access, and preventive service use among noninstitutionalized adults aged 18 years or older in the United States.

BRFSS data are based on self-reported telephone interviews and, since 2011, have been collected using both landline and cellular telephones. This dual-frame design improves population coverage and representativeness as telephone use patterns have shifted over time. Survey data are collected by state health departments following CDC protocols, then sent to the CDC for editing, weighting, and public release.

To improve representativeness, BRFSS uses iterative proportional fitting (“raking”) to generate survey weights. This method adjusts the sample to better reflect the demographic makeup of each state using characteristics such as age, sex/gender, race and ethnicity, education, marital status, home ownership, geographic region, and telephone source.

Across years, BRFSS follows a common structure that includes a core questionnaire, optional modules, and state-added questions. The core questionnaire is administered across participating states, while optional modules may vary by year and location. By combining multiple annual BRFSS datasets, this study uses a large, well-established public health surveillance source while accounting for modest year-to-year differences in survey content and participation.

Raw data acquisition

The raw input files are the annual BRFSS public-use SAS transport files released by the CDC. I worked from the 2020 through 2024 annual survey cycles and imported the .XPT files with haven::read_xpt().

Variables retained and harmonized

The main report focuses on frequent mental distress plus a set of geography, time, demographic, socioeconomic, behavioral, and psychosocial variables. Only those variables were carried forward into the analytic dataset.

Three variables had to be explicitly harmonized across cycles before binding the files together:

  • _INCOMG in 2020 was copied into _INCOMG1 to match the 2021+ naming convention.
  • MEDCOST in 2020 was copied into MEDCOST1.
  • DRNKANY5 in 2020-2021 was copied into DRNKANY6 to align with the 2022+ variable name.

Income also needed a content harmonization, not just a rename. The 2020 BRFSS income field used a 5-level scheme, while later years split the top category into multiple higher-income bins. To keep the cross-year comparisons valid, I collapsed every cycle into a shared 5-level scale: <$15k, $15-25k, $25-35k, $35-50k, and $50k+.

Cleaning and recoding

The preparation script followed a small set of consistent rules before saving the analytic file:

  1. Read each yearly .XPT file and retain only the variables needed for the report.
  2. Add a year column and standardize renamed variables so all years share one schema.
  3. Remove U.S. territories by keeping only the 50 states plus the District of Columbia.
  4. Map _STATE FIPS codes to full state names and two-letter abbreviations.
  5. Convert BRFSS special missing codes to NA, while converting MENTHLTH == 88 to 0 poor-mental-health days.
  6. Define frequent mental distress as _MENT14D == 3, matching the CDC’s 14-day threshold.
  7. Recode selected variables into labelled factors for plotting and tabulation.
  8. Save the cleaned analytic dataset as brfss_subset.rds.

Important recoding notes:

  • Tennessee is absent from the 2024 public data because it did not meet minimum reporting requirements that year.
  • EMPLOY1 == 7 (“Retired”) is a valid category and was kept as such.
  • The sleep variable SLEPTIM1 was originally considered for analysis, but it is not available in the 2024 core file, so sleep analyses were dropped from the final report.

Distribution of poor mental health days

Show code 
mental_days_dist <- base_2024 |>
  filter(!is.na(mental_days)) |>
  count(mental_days) |>
  mutate(
    group = case_when(
      mental_days == 0  ~ "0 days",
      mental_days == 30 ~ "30 days",
      mental_days >= 14 ~ "14+ days",
      TRUE              ~ "1-13 days"
    ),
    group = factor(group, levels = c("0 days", "1-13 days", "14+ days", "30 days"))
  )

plot_ly(
  mental_days_dist,
  x = ~mental_days, y = ~n, color = ~group,
  colors = c("0 days" = "#27ae60", "1-13 days" = "#bdc3c7",
             "14+ days" = "#e67e22", "30 days" = "#c0392b"),
  type = "bar",
  hovertemplate = "%{x} days<br>Respondents: %{y:,}<extra></extra>"
) |>
  layout(
    title = list(text = "Exploratory check: poor mental health days in 2024", x = 0),
    xaxis = list(title = "Days of poor mental health in the last 30 days", dtick = 5),
    yaxis = list(title = "Respondents"),
    legend = list(title = list(text = "Category"), orientation = "h",
                  x = 0.5, xanchor = "center", y = -0.2),
    barmode = "stack",
    margin = list(t = 60, b = 80)
  )

This distribution is one reason the dichotomized FMD outcome is useful here: respondents cluster heavily at 0 days and again at the extreme upper tail, rather than forming a smooth continuous distribution.
